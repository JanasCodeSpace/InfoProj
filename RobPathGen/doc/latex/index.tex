\chapter{Beschreibung Roboter Path Editor}
\hypertarget{index}{}\label{index}\index{Beschreibung Roboter Path Editor@{Beschreibung Roboter Path Editor}}
\hypertarget{index_intro_sec}{}\doxysection{\texorpdfstring{Nutzen}{Nutzen}}\label{index_intro_sec}
Mit diesem Programm sollen haendisch aufgenommene Pfad Daten einer Roboterbewegung zu einem fuer Kuka Roboter lesbaren File gemacht werden. Zusaetzlich soll einstellbar sein ob die Orientierung berechnet werden soll, oder eingegeben werden soll. Das selbe gilt fuer Geschwindigkeitsdaten.\hypertarget{index_install_sec}{}\doxysection{\texorpdfstring{Aufbau}{Aufbau}}\label{index_install_sec}
In der Grundidee werden die eingelesenen Daten immer aus der vorhergegangenen Klasse ausgelesen und nach der Verarbeitung in der aktuellen Klasse gespeichert.\hypertarget{index_step1}{}\doxysubsection{\texorpdfstring{Logging}{Logging}}\label{index_step1}
Zuerst wird die Loggingklasse \doxylink{class_c_logging}{CLogging} initialisiert. In ihr wird gespeichert in welchem Schritt das Programm gerade ist. Dieser Klasse wird ein Pfad uebergeben an welchem die Daten gespeichert werden sollen.\hypertarget{index_step2}{}\doxysubsection{\texorpdfstring{Daten einlesen}{Daten einlesen}}\label{index_step2}
Als naechstes werden die Nutzerdaten eingelesen und anschliessend die aufgenommenen Daten eingelesen. Dabei wird Ueberprueft ob es sich um einen zusammenhaengenden Pfad handelt. Das passiert in der Klasse \doxylink{class_c_input_parameter}{CInput\+Parameter}.\hypertarget{index_step3}{}\doxysubsection{\texorpdfstring{Daten verarbeiten}{Daten verarbeiten}}\label{index_step3}
In mehreren Schritten folgt eine Nachbearbeitung der Daten. Zuerst werden die Daten mit einem gleitendem Mittelwertfilter in der Klasse \doxylink{class_c_mean_filter}{CMean\+Filter} geglaettet. Anschliessend werden Punkte mit Hilfe des Douglas-\/\+Peuker Algorithmuses in der Klasse \doxylink{class_c_segment_approximator}{CSegment\+Approximator} geloescht. Sollten es mehrere nicht zusammenhaengende Pfade sein muessen diese jetzt noch zusammengesetzt werden.\hypertarget{index_step4}{}\doxysubsection{\texorpdfstring{Roboter Code erstellen}{Roboter Code erstellen}}\label{index_step4}
Als letzter Schritt werden die Nutzereinstellungen in die Daten uebernommen und der Robotercode erstellt.
